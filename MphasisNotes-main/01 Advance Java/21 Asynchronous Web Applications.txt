
# Asynchronous Web Applications

---

## 1. What is an Asynchronous Web Application?

### Definition

> An asynchronous web application allows the browser and server to communicate **without blocking the page**.

The page does not reload.
User can continue interacting.

This creates:

✔ smooth UI
✔ real-time updates
✔ faster experience

---

## Synchronous vs Asynchronous

### Traditional (synchronous)

```
User click
  ↓
Browser waits
  ↓
Server responds
  ↓
Page reload
```

User cannot do anything while waiting.

---

### Asynchronous

```
User click
  ↓
JS sends request
  ↓
User continues using page
  ↓
Response arrives in background
  ↓
UI updates dynamically
```

👉 No page reload

This is how Gmail, WhatsApp Web, dashboards work.

---

## 2. Using the Asynchronous Servlet Mechanism

Servlets normally block a thread while processing.

Asynchronous servlets allow:

> server thread to be released while work continues

This improves scalability.

---

## Why async servlet is needed

Without async:

* each request holds a thread
* slow operations waste resources

With async:

* servlet starts background task
* thread returns to pool
* response completed later

👉 more concurrent users supported

---

## Basic Async Servlet Example

```java
@WebServlet(value="/async",
            asyncSupported=true)
public class AsyncServlet extends HttpServlet {

    protected void doGet(HttpServletRequest req,
                         HttpServletResponse resp)
            throws IOException {

        AsyncContext ctx = req.startAsync();

        ctx.start(() -> {
            try {
                Thread.sleep(3000); // simulate work
                resp.getWriter()
                    .write("Async response ready");
                ctx.complete();
            } catch(Exception e) {}
        });
    }
}
```

---

### What happens

1. Request arrives
2. Servlet starts async mode
3. Background task runs
4. Response sent later

User does not block server thread.

## 3. Using JavaScript to Send an HTTP Request

This is client-side async.

We use:

👉 AJAX / Fetch API

---

## Example: Fetch API

```javascript
fetch("/async")
  .then(response => response.text())
  .then(data => {
      document.getElementById("msg").innerText = data;
  });
```

This sends request in background.

No page reload.

---

## What is happening

```
JS → HTTP request → servlet
     ← HTTP response ←
JS updates page
```

User never sees reload.

---

## 4. Processing Response Entirely in JavaScript

Instead of JSP rendering HTML:

👉 server sends raw data (JSON/text)
👉 JavaScript builds UI

Example:

### Server returns JSON

```
{ "name": "Amit", "age": 25 }
```

### JavaScript handles it

```javascript
fetch("/user")
 .then(res => res.json())
 .then(user => {
    document.getElementById("name")
      .innerText = user.name;
 });
```

👉 UI built in browser

This is modern SPA behavior.

---

## Why this matters

* reduces server rendering
* faster UI updates
* rich client apps

---

## 5. Combining Techniques → Server Push Effect

We combine:

✔ async servlet
✔ JS async request
✔ JS UI update

to simulate real-time push.

---

## How server-push effect works

Browser repeatedly asks:

```
"Any update?"
"Any update?"
"Any update?"
```

This is called:

👉 long polling / async polling

---

## Example Concept

```javascript
function poll() {
  fetch("/async")
    .then(res => res.text())
    .then(data => {
        updateUI(data);
        poll(); // repeat
    });
}
poll();
```

Server responds when data ready.

Feels like push.

## Real technologies that evolved from this

* WebSockets

* chat apps

Async servlet is foundation.

## Synchronous vs Asynchronous Summary

| Feature     | Synchronous | Asynchronous |
| ----------- | ----------- | ------------ |
| Blocking    | Yes         | No           |
| Page reload | Yes         | No           |
| User waits  | Yes         | No           |
| Scalability | Lower       | Higher       |
| UX          | Basic       | Modern       |

---

## Memory Rules

> async = non-blocking
> JS handles UI
> servlet handles data
> no reload = modern app

---

## Key Takeaways

* async apps feel real-time
* servlet async improves scalability
* JavaScript handles background communication
* UI updates without reload
* foundation of modern web apps

## One-line summary

> Asynchronous apps allow background communication without freezing the UI.
