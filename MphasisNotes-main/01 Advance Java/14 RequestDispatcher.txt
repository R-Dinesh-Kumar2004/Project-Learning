
# RequestDispatcher.forward() vs RequestDispatcher.include()

---

## 1. What is RequestDispatcher?

### Definition

> **RequestDispatcher** is used for **server-side communication between resources**.

Resources can be:

* Servlets
* JSPs
* HTML files

👉 Navigation happens **inside the server**, browser is **not aware**.

---

### ✅ Why do we need RequestDispatcher? (Use Case)

* To divide application logic into multiple components
* To reuse code
* To share request data between resources

Example:

* One servlet handles validation
* Another servlet handles response

---

## 2. How to Get RequestDispatcher

```java
RequestDispatcher rd =
        request.getRequestDispatcher("home.jsp");
```

---

### ✅ Why this is necessary?

* Server must know **which internal resource** to forward/include
* Browser cannot decide this

---

## 3. RequestDispatcher.forward()

---

### 3️⃣.1 What is `forward()`?

> `forward()` transfers control from one resource to another **on the server**.

* Same request object
* Same response object
* No new request is created

---

### 3️⃣.2 What Happens Internally?

1. Client sends request
2. First servlet processes request
3. Servlet calls `forward()`
4. Control moves to another resource
5. Response is generated

👉 **Only one request is involved**

---

### 3️⃣.3 Key Characteristics

* Request is forwarded
* URL in browser **does not change**
* Request data is preserved

---

### ✅ Why do we need `forward()`? (Use Case)

* Share request data
* Implement MVC architecture
* Avoid duplicate requests
* Internal navigation

Example:

* LoginServlet → DashboardServlet
* Controller → JSP (view)

---

### 3️⃣.4 Example

```java
RequestDispatcher rd =
        request.getRequestDispatcher("dashboard.jsp");
rd.forward(request, response);
```

---

## 4. RequestDispatcher.include()

---

### 4️⃣.1 What is `include()`?

> `include()` includes output of another resource **inside the current response**.

* Same request
* Same response
* Output is combined

---

### 4️⃣.2 What Happens Internally?

1. Client sends request
2. Servlet writes partial response
3. `include()` is called
4. Another resource writes output
5. Control returns to original servlet

👉 Output is **merged**

---

### 4️⃣.3 Key Characteristics

* Original response continues
* Included resource cannot control full response
* Useful for reusable UI parts

---

### ✅ Why do we need `include()`? (Use Case)

* Reuse common content
* Avoid duplication
* Build page fragments

Examples:

* Header
* Footer
* Menu bar

---

### 4️⃣.4 Example

```java
RequestDispatcher rd =
        request.getRequestDispatcher("header.jsp");
rd.include(request, response);
```

---

## 5. forward() vs include() – Comparison Table

| Feature          | forward()   | include()        |
| ---------------- | ----------- | ---------------- |
| Type             | Server-side | Server-side      |
| Requests         | One         | One              |
| URL changes      | No          | No               |
| Request data     | Preserved   | Preserved        |
| Response control | Transferred | Shared           |
| Output           | Replaced    | Appended         |
| Typical use      | Navigation  | Reusable content |

---

## 6. forward/include vs sendRedirect (Quick Contrast)

| Feature      | forward/include | sendRedirect |
| ------------ | --------------- | ------------ |
| Side         | Server          | Client       |
| Requests     | One             | Two          |
| URL change   | No              | Yes          |
| Request data | Available       | Lost         |

---

## 7. Common Student Mistakes (and WHY)

❌ Expecting URL change with forward
👉 Browser never sees forward

❌ Writing output before forward
👉 Response may already be committed

❌ Using redirect for internal navigation
👉 Redirect creates unnecessary new request

---

## 8. One-Line Memory Rules (Very Effective)

* **forward()** → move control
* **include()** → reuse output
* **Both** → server-side
* **Redirect** → client-side

---

## Key Takeaways

* RequestDispatcher works inside server
* forward() transfers control
* include() merges output
* Both preserve request data
* Essential for MVC design
