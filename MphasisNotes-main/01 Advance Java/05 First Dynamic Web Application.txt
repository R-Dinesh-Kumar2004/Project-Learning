## 5. WEB-INF Folder (Protected Area)

### Rule

=> Files inside `WEB-INF` *cannot be accessed directly from browser*.

Contents:

* `web.xml`
* compiled classes
* libraries

рџ‘‰ This protects internal configuration.

---

## 6. What is `web.xml` (Deployment Descriptor)?

### Simple Definition

=> `web.xml` is a configuration file that tells the server *how the web application is structured*.

It contains:

* servlet mappings
* welcome files
* filters
* context parameters

рџ‘‰ Server *reads this file at startup*

## 8. Step-by-Step Deployment Flow (Tomcat)

(Example server: *Apache Tomcat*)

---

### Step 1. Application Deployment

* We run the project on server
* Application is deployed to Tomcat

---

### Step 2. Tomcat Reads `web.xml`

Tomcat:

* reads `web.xml`
* loads configuration data

This includes:

* welcome files
* servlet mappings
* filters
* context parameters

---

### Step 3. Welcome File Resolution

Tomcat:

* checks welcome file list
* finds `index.html` or configured file
* prepares it for first request

---

### Step 4. URL Pattern Scanning

Tomcat:

* scans all servlet URL mappings
* builds internal mapping table

Example:

```text
/login в†’ LoginServlet
```

рџ‘‰ This is how Tomcat knows *which servlet to call*

---

### Step 5. Configuration & Context Data

Tomcat:

* reads server config
* reads application-level parameters
* prepares shared data

---

### Step 6. ServletContext Creation (Application Level)

### What is ServletContext?

=> An object created by Tomcat *per web application*.

* Created *only once*
* Shared across all servlets
* Stores application-wide data

рџ‘‰ Tomcat:

* creates `ServletContext`
* fills it with configuration data

---

### Step 7. ServletConfig Creation (Servlet Level)

### What is ServletConfig?

=> An object created *per servlet*.

* Contains servlet-specific data
* Not shared with other servlets

рџ‘‰ Tomcat creates:

* one `ServletConfig` per servlet
* attaches it to servlet instance

---

### Step 8. Registry Maintenance

Tomcat maintains:

* list of servlets
* URL mappings
* lifecycle status

рџ‘‰ This internal registry helps Tomcat route requests quickly.

---

### Step 9. First Request Handling

* Browser sends request `/`
* Tomcat serves:

  * welcome file (HTML), or
  * mapped servlet (if configured)

рџ‘‰ Application is now *live*
