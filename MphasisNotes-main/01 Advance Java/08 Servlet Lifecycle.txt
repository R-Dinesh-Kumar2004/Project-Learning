
# Servlet Lifecycle & Internal Working of Tomcat

---

## 1.  What is Servlet Lifecycle?

### Simple Definition

> **Servlet lifecycle describes the stages a servlet goes through from creation to destruction.**

Lifecycle is **managed by the server**, not by us.

đź‘‰ We **do not create servlet objects manually**.

---

## 2.  Who Manages Servlet Lifecycle?

* Lifecycle is managed by the web container
* Example container: **Apache Tomcat**

đź‘‰ Tomcat:

* creates servlet objects
* initializes them
* handles requests
* destroys them

---

## 3.  Servlet Lifecycle Phases (High Level)

A servlet goes through **three main phases**:

1. **Loading & Instantiation**
2. **Initialization**
3. **Request Processing**
4. **Destruction**

### Step 7. `init()` Method is Called

## 4.5 Request Processing Phase (MOST IMPORTANT)

This phase happens **for every request**.

---

### Step 8. Client Sends HTTP Request

* Browser sends request
* Example:

  * GET
  * POST

---

### Step 9. Request & Response Objects are Created

Tomcat:

* creates a **new HttpServletRequest object**
* creates a **new HttpServletResponse object**

đź‘‰ These objects are:

* request-specific
* created **for every request**
* destroyed after response is sent

---

### Step 10. `service()` Method is Called

```java

                    HttpServletResponse res)
```

> `service()` is the **central method**.

Tomcat:

* calls `service()`
* passes request and response objects

---

### Step 1. 1.  â€“ `service()` Calls `doGet()` / `doPost()`

Inside `service()`:

* Tomcat checks HTTP method
* Calls appropriate method:

| HTTP Method | Method Called |
| ----------- | ------------- |
| GET         | `doGet()`     |
| POST        | `doPost()`    |
| PUT         | `doPut()`     |
| DELETE      | `doDelete()`  |

đź‘‰ **We override `doGet()` / `doPost()`**
đź‘‰ **We never call `service()` ourselves**

### Step 1. 2.  â€“ Response is Sent Back

## 6.  Lifecycle Methods Summary Table

| Method           | Called When     | How Many Times |
| ---------------- | --------------- | -------------- |
| Constructor      | Object creation | Once           |
| init()           | Initialization  | Once           |
| service()        | Every request   | Many           |
| doGet()/doPost() | Based on method | Many           |
| destroy()        | Shutdown        | Once           |

---

## 7.  Key Rules (Very Important)

* Servlet object is **single**
* Requests are **multiple**
* Request & response objects are **new every time**
* ServletContext is **application-wide**
* ServletConfig is **servlet-specific**
